System Overview, Roles & Login

Gambaran Sistem, Peran & Login

BSpace Finance is a daily reconciliation system that matches POS cashier data against bank mutation statements to detect missing transfers, amount mismatches, and unexplained bank credits.

BSpace Finance adalah sistem rekonsiliasi harian yang mencocokkan data kasir POS dengan mutasi bank untuk mendeteksi transfer yang tidak ditemukan, selisih jumlah, dan kredit bank yang tidak terduga.

🔐 Access the App Cara Akses Aplikasi

http://68.183.229.3:3001/login
  • Use your work email and password provided by the Administrator. Gunakan email kerja dan password yang diberikan oleh Administrator.
  • After login you land on the Dashboard — it shows recent sessions and key metrics. Setelah login, Anda masuk ke Dashboard — menampilkan sesi terbaru dan metrik utama.
  • The notification bell (top-right) shows sessions waiting for your attention. Lonceng notifikasi (kanan atas) menampilkan sesi yang menunggu tindakan Anda.

Workflow 1 — Create a New Reconciliation Session

Alur 1 — Buat Sesi Rekonsiliasi Baru

This is done once per day per outlet, usually by Finance Staff at end-of-day or the morning after. You'll upload the cashier Excel file and bank mutation files for all banks used that day.

Ini dilakukan sekali per hari per outlet, biasanya oleh Finance Staff di akhir hari atau pagi harinya. Anda akan mengupload file Excel kasir dan file mutasi bank untuk semua bank yang digunakan hari itu.

📋 What You Need Before Starting Yang Perlu Disiapkan Sebelum Mulai

  • Cashier file: TEMPLATE_KASIR_FINSETTLE_v3_final.xlsx (or compatible format) — exported from the POS/cashier system File kasir: TEMPLATE_KASIR_FINSETTLE_v3_final.xlsx (atau format yang kompatibel) — diekspor dari sistem POS/kasir
  • Bank mutation files (.xlsx, .xls, or .csv) for BCA, BNI, BRI, MANDIRI — downloaded from each bank's internet banking portal File mutasi bank (.xlsx, .xls, atau .csv) untuk BCA, BNI, BRI, MANDIRI — diunduh dari portal internet banking masing-masing bank

Jika ada kesalahan, hapus sesi yang ada terlebih dahulu (lihat Alur 7).

Alur 2 — Jalankan Matching & Tinjau Hasil

After a session is created, you run the reconciliation engine. It compares each EDC cashier entry against the bank mutations and produces a match result for every entry.

Setelah sesi dibuat, jalankan engine rekonsiliasi. Engine akan membandingkan setiap entri EDC kasir dengan mutasi bank dan menghasilkan hasil matching untuk setiap entri.

📊 Understanding the Review Page Layout Memahami Tampilan Halaman Review

  • REG block always appears first, EV block below. Each block is followed by its Ringkasan (summary by cashier). Blok REG selalu muncul pertama, blok EV di bawahnya. Setiap blok diikuti Ringkasan (ringkasan per kasir).
  • Each bank is shown as a separate card within the block. The Total column (light gray background) shows the sum across all cashiers. Setiap bank ditampilkan sebagai kartu terpisah dalam blok. Kolom Total (latar abu-abu muda) menunjukkan jumlah semua kasir.
  • Status Bank column: Cocok = matched, Tidak Ada di Bank = no matching mutation found, Nol/Lewati = zero amount or CASH/VOUCHER. Kolom Status Bank: Cocok = cocok, Tidak Ada di Bank = tidak ada mutasi yang cocok, Nol/Lewati = jumlah nol atau CASH/VOUCHER.
  • Mutasi Tak Terduga section (outside the main card): bank transactions that have no matching cashier entry. These are informational — no action required unless suspicious. Bagian Mutasi Tak Terduga (di luar kartu utama): transaksi bank yang tidak ada entri kasirnya. Bersifat informatif — tidak perlu tindakan kecuali mencurigakan.
  • Use the Tabs (Semua Entri / Perlu Perhatian / Cocok Saja) to filter the view. Gunakan Tab (Semua Entri / Perlu Perhatian / Cocok Saja) untuk menyaring tampilan.
Tip: Hover over any truncated text in the Entitas column to see the full description in a tooltip. Arahkan mouse ke teks yang terpotong di kolom Entitas untuk melihat deskripsi lengkap dalam tooltip.

Workflow 4 — Add a Forgotten Bank Mutation File

Alur 4 — Tambah File Mutasi Bank yang Terlewat

If you discover that you forgot to upload one of the bank mutation files during session creation, you can add it directly from the Review page without deleting and redoing the entire session.

Jika Anda menemukan bahwa ada file mutasi bank yang terlewat saat membuat sesi, Anda bisa menambahkannya langsung dari halaman Review tanpa harus menghapus dan mengulang seluruh sesi.

System
Appends Mutations — Never Replaces
New mutations are added to the existing ones. Previously uploaded bank data is preserved.
System — Auto
Matching Re-runs Automatically
After upload, reconciliation engine reruns. Previously unmatched entries may now find a match.
Done
Review Updated Results
Summary cards refresh. Check if the previously unmatched entries are now resolved.
Finance Staff
Di halaman Review, klik "Tambah File Mutasi"
Tombol terlihat ketika sesi belum ditandatangani
Finance Staff
Pilih Nama Bank & Pilih File
Pilih bank dari dropdown (BCA / BNI / BRI / MANDIRI). Upload satu atau lebih file mutasi untuk bank tersebut.
Sistem

Mutasi baru ditambahkan ke yang sudah ada. Data bank yang sudah diupload sebelumnya tetap tersimpan.

Jika ingin mengganti file mutasi bank sepenuhnya (file yang salah diupload), Anda perlu menghapus sesi dan memulai ulang. Tambah selalu bersifat menambahkan.

Workflow 7 — Delete & Redo a Session

Alur 7 — Hapus & Ulangi Sesi

If a session was created with the wrong date, wrong files, or wrong block type, it can be deleted and recreated. Only use this as a last resort — prefer adding mutation files (Workflow 4) if the issue is just a missing bank file.

Jika sesi dibuat dengan tanggal salah, file salah, atau tipe blok salah, sesi bisa dihapus dan dibuat ulang. Gunakan ini sebagai opsi terakhir — lebih baik tambahkan file mutasi (Alur 4) jika masalahnya hanya file bank yang kurang.

📞 Need Help? Butuh Bantuan?

  • If a bank file fails to parse (0 mutations found), the system will suggest an AI-powered column configuration fix. Accept or modify the suggestion, then retry. Jika file bank gagal dibaca (0 mutasi ditemukan), sistem akan menyarankan perbaikan konfigurasi kolom berbasis AI. Terima atau modifikasi saran tersebut, lalu coba lagi.
  • If a session is stuck or showing unexpected data, contact your Administrator who can check the audit log and reset the session. Jika sesi macet atau menampilkan data yang tidak terduga, hubungi Administrator Anda yang bisa memeriksa log audit dan mereset sesi.
  • For suspicious "Tidak Ada di Bank" entries that cannot be explained, escalate immediately — do not ignore without documentation. Untuk entri "Tidak Ada di Bank" yang mencurigakan dan tidak bisa dijelaskan, eskalasi segera — jangan diabaikan tanpa dokumentasi.
